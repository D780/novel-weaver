# Data Agent 实体自动提取

## 系统原理

Data Agent 在写作过程中自动提取故事中的实体（角色、物品、地点、势力等），
并自动消歧、入库管理，确保所有实体都有唯一记录。

---

## 一、实体类型

### 1. 角色实体 (Character)

| 属性 | 类型 | 说明 | 示例 |
|------|------|------|------|
| id | String | 唯一标识符 | "C001", "C002" |
| name | String | 角色名 | "叶凡", "萧炎" |
| alias | String[] | 别名/称号 | ["叶天帝", "荒天帝"] |
| role | Enum | 角色类型 | 主角/配角/反派/龙套 |
| gender | Enum | 性别 | 男/女/未知 |
| age | Number | 年龄（故事内） | 18, 200 |
| status | Enum | 当前状态 | 活跃/死亡/失踪/沉睡 |
| faction | String | 所属势力 | "荒古世家", "宗门名" |
| power_level | String | 当前等级 | "炼气3层", "筑基初期" |
| personality | String[] | 性格特征 | ["谨慎", "果断"] |
| relationships | Object | 人际关系 | {"父亲": "角色ID", "师傅": "角色ID"} |
| items | String[] | 持有物品ID | ["I001", "I002"] |
| abilities | String[] | 掌握能力ID | ["A001"] |
| first_appearance | String | 首次出场章节 | "第3章" |
| last_appearance | String | 最近出场章节 | "第25章" |

## 二、实体提取流程

### 自动提取

```
1. AI 生成章节内容
   ↓
2. NLP 识别实体：
   - 新角色名 → 创建角色实体
   - 新物品名 → 创建物品实体
   - 新地点名 → 创建地点实体
   - 新势力名 → 创建势力实体
   ↓
3. 实体消歧：
   - 同义词识别（别名/简称）
   - 同名异人区分
   - 实体合并
   ↓
4. 分配唯一 ID：
   - 角色：C001, C002...
   - 物品：I001, I002...
   - 地点：L001, L002...
   - 势力：F001, F002...
   - 能力：A001, A002...
   - 事件：E001, E002...
   ↓
5. 入库管理：
   - 添加到 truth-files/entities.md
   - 关联到角色档案
   - 关联到世界观
   ↓
6. 作者确认：
   - AI 提示新实体
   - 作者确认/修改/删除
```

### 手动管理

```
/nw entity list 列出所有实体
/nw entity type 列出指定类型实体
/nw entity show [ID] 显示实体详情
/nw entity add 手动添加实体
/nw entity update 更新实体
/nw entity delete 删除实体
/nw entity merge 合并重复实体
```

## 三、实体消歧规则

### 同义词识别

```
叶凡 = 叶天帝 = 荒天帝 → 合并为 C001（叶凡）
神秘玉佩 = 传承玉佩 = 古老玉佩 → 合并为 I001（神秘玉佩）
```

### 同名区分

```
"萧炎"（斗破苍穹）= C001
"萧炎"（同人）= C100
→ 根据上下文区分
```

### 实体变更追踪

```
每次实体状态变更都记录：
{
  "id": "C001",
  "changes": [
    {
      "field": "power_level",
      "old": "炼气3层",
      "new": "炼气4层",
      "chapter": "第15章",
      "reason": "突破"
    }
  ]
}
```

## 五、实体关系图谱

```
C001（叶凡）
├── 父亲 → C010
├── 师傅 → C011
```

## 六、使用场景

### 写作前

```
/nw entity list 角色  → 查看有哪些角色
/nw entity show C001 → 查看主角详细信息
```

### 写作中

```
AI 自动提取新实体 → 提示作者确认
作者确认 → 自动入库
```

### 写作后

```
/nw entity list 未使用 → 查看未使用的实体
/nw entity merge → 合并重复实体
/nw entity report → 生成实体报告
```

### 查找问题

```
/nw entity orphan → 查找孤立实体（无关联）
/nw entity conflict → 查找实体冲突
/nw entity timeline → 查看实体时间线
```
