# 记忆系统详解

## 三重记忆架构

NovelWeaver 采用三重记忆机制，确保AI在长篇写作中不忘事、不混乱。

```
┌─────────────────────────────────────────┐
│           三重记忆架构                    │
├─────────────────────────────────────────┤
│ 第一层：真相文件（Truth Files）           │
│  - 持久化的世界状态和设定                 │
│  - 手动更新+自动同步                      │
│  - 相当于"设定集"                         │
├─────────────────────────────────────────┤
│ 第二层：长程上下文（Context）             │
│  - 前文摘要和相关章节                     │
│  - 写作时自动检索注入                     │
│  - 相当于"前情提要"                       │
├─────────────────────────────────────────┤
│ 第三层：实体关系（Entity Graph）          │
│  - 角色关系、地点关联、物品追踪           │
│  - 自动维护和更新                         │
│  - 相当于"人物关系图"                     │
└─────────────────────────────────────────┘
```

---

## 第一层：真相文件 (Truth Files)

真相文件是持久化的设定文档，由作者和AI共同维护。

### 文件清单

| 文件名 | 内容 | 更新频率 |
|--------|------|---------|
| `current-state.md` | 世界当前状态 | 每章后 |
| `characters.md` | 角色档案 | 角色变化时 |
| `world-setting.md` | 世界观设定 | 新设定时 |
| `pending-hooks.md` | 待处理伏笔 | 每章后 |
| `power-system.md` | 力量体系 | 体系变化时 |

### current-state.md 模板

```markdown
# 世界当前状态

## 时间线
- 当前时间：[故事内时间]
- 距离开篇：[时间跨度]

## 主角状态
- 位置：[当前地点]
- 境界/等级：[当前等级]
- 持有物品：[重要物品列表]
- 当前目标：[短期目标]

## 活跃剧情线
- 主线：[主线剧情进展]
- 支线1：[支线剧情进展]
- 支线2：[支线剧情进展]

## 近期事件
- [最近发生的重要事件]

## 待解决冲突
- [尚未解决的矛盾]
```

### characters.md 模板

```markdown
# 角色档案

## [角色名]
- **身份**：[角色身份]
- **性格**：[性格特点]
- **能力**：[拥有的能力]
- **关系**：[与其他角色的关系]
- **当前状态**：[当前位置/状态]
- **目标**：[角色目标]
- **秘密**：[隐藏信息]
- **成长轨迹**：[角色变化历程]
```

### pending-hooks.md 模板

```markdown
# 伏笔追踪表

| 伏笔 | 设置章节 | 预期回收 | 状态 | 备注 |
|------|---------|---------|------|------|
| [伏笔描述] | 第X章 | 第Y章 | 未回收 | [说明] |
```

## 第二层：长程上下文 (Context)

长程上下文是AI写作时自动检索的相关信息。

### 上下文构建规则

写作第N章时，自动注入：

1. **前3章摘要** - 最近发生的事情
2. **相关角色档案** - 本章出场角色的设定
3. **相关伏笔** - 与本章情节相关的伏笔
4. **世界状态** - 当前的世界状态
5. **大纲参考** - 本章在大纲中的定位

### 上下文注入示例

```markdown
【写作参考】

## 前情摘要
- 第N-3章：[摘要]
- 第N-2章：[摘要]
- 第N-1章：[摘要]

## 本章出场角色
- [角色A]：[关键设定]
- [角色B]：[关键设定]

## 相关伏笔
- [伏笔1]：[描述]，设置于第X章，待回收

## 当前世界状态
- 时间：[时间]
- 地点：[地点]
- 状态：[状态]

## 本章大纲目标
[本章需要完成的情节目标]
```

## 第三层：实体关系 (Entity Graph)

实体关系是角色、地点、物品之间的关联网络。

### 关系类型

| 关系类型 | 说明 | 示例 |
|---------|------|------|
| 人物关系 | 角色之间的关联 | 师徒、父子、仇敌 |
| 地点关联 | 地点之间的关联 | 相邻、从属 |
| 物品归属 | 物品的所有者 | 谁拥有某物品 |
| 事件关联 | 事件的因果关联 | A事件导致B事件 |

### 关系维护规则

1. **新角色出场** - 记录与其他角色的关系
2. **关系变化** - 更新关系状态
3. **物品转移** - 更新物品归属
4. **地点变化** - 更新角色位置

---

## 记忆更新流程

### 写前准备

1. 读取真相文件
2. 检索前文摘要
3. 加载相关角色档案
4. 构建写作上下文

### 写后更新

1. 更新当前世界状态
2. 生成章节摘要
3. 更新伏笔状态
4. 更新角色状态
5. 保存真相文件

## AI 自动检索规则（RAG 增强）

本节定义 AI 在执行不同指令时应自动检索的内容范围。AI 应主动执行检索，无需用户指定文件。

### 检索优先级

AI 在检索时应按以下优先级顺序：

```
优先级 1：真相文件（truth-files/）→ 当前设定的权威来源
优先级 2：大纲文件（novels/outline.md、volume-XX/plan.md）→ 剧情目标
优先级 3：前文摘要（.novel-weaver/summaries/）→ 最近剧情回顾
优先级 4：相关章节（novels/volume-XX/chapters/）→ 细节参考
优先级 5：文风库（styles/）→ 风格参考
```

### 指令驱动的检索矩阵

| 用户意图关键词 | 应检索的文件 | 检索目标 |
|--------------|-------------|---------|
| 写/继续/下一章 | truth-files/* + outline.md + 前3章 | 角色设定、世界观、力量体系、前情、大纲目标 |
| 主角/角色/人物 | truth-files/characters.md + 相关章节 | 角色档案、出场历史、成长轨迹 |
| 等级/修炼/力量 | truth-files/power-system.md + current-state.md | 力量体系规则、当前等级 |
| 地点/世界/设定 | truth-files/world-setting.md + current-state.md | 世界观设定、当前地点 |
| 伏笔/悬念/回收 | truth-files/pending-hooks.md + 相关章节 | 未回收伏笔、设置位置 |
| 检查/矛盾/一致 | truth-files/* + 被检查章节 | 全部设定+前文，做一致性对比 |
| 规划/大纲/计划 | outline.md + truth-files/* + 已写卷 | 总大纲、当前设定、已有内容 |
| 灵感/建议/走向 | truth-files/* + outline.md + 前文 | 当前剧情线、未解冲突、大纲目标 |

### 语义检索规则

AI 不应仅依赖文件名，而应理解用户意图后检索相关内容：

1. **角色相关查询**：
   - 用户问"主角现在什么等级"→ 检索 `characters.md` + `current-state.md` + `power-system.md`
   - 用户说"写主角遇到师父"→ 检索 `characters.md` 找"师父"相关角色 + 前文出场历史

2. **设定相关查询**：
   - 用户问"这个世界有什么力量"→ 检索 `power-system.md` + `world-setting.md`
   - 用户说"主角突破到下一境界"→ 检索 `power-system.md` 确认境界名称和规则

3. **剧情相关查询**：
   - 用户问"之前埋了什么伏笔"→ 检索 `pending-hooks.md` + 相关章节
   - 用户说"继续上一章的剧情"→ 检索前一章全文 + 前情摘要

### 检索上下文构建模板

AI 检索后，应按以下结构构建写作上下文（内部使用，无需展示给用户）：

```markdown
【写作上下文】

## 角色状态（来自 truth-files/characters.md + current-state.md）
- [角色A]：当前等级/位置/状态/目标
- [角色B]：当前等级/位置/状态/目标

## 世界设定（来自 truth-files/world-setting.md + power-system.md）
- 相关世界观：[与本章相关的设定]
- 力量体系：[本章涉及的境界/能力规则]

## 前情摘要（来自 .novel-weaver/summaries/ + 前3章）
- 最近事件：[前3章关键情节]
- 未解冲突：[尚未解决的矛盾]
- 待回收伏笔：[相关伏笔列表]

## 本章目标（来自 outline.md / volume-XX/plan.md）
- 大纲要求：[本章应完成的情节]
- 角色成长：[应推进的角色发展]
- 伏笔处理：[应埋设/回收的伏笔]
```

### 检索失败处理

如果 AI 无法找到相关文件（如文件不存在或内容为空）：

1. **首次写作**：真相文件可能未初始化 → 提示用户先执行 `/nw init`
2. **缺少设定**：用户询问不存在的角色/设定 → 询问用户是否要创建新设定
3. **前文缺失**：找不到前文 → 告知用户这是第一章，无需前情参考

## 使用建议

### 新手

- 保持真相文件简洁
- 每章后更新摘要
- 定期整理伏笔表

### 进阶

- 细化角色档案
- 建立详细的时间线
- 维护力量体系文档

### 专家

- 自定义真相文件结构
- 建立完整的实体关系图
- 使用多线索追踪系统
