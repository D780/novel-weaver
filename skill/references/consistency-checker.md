# Consistency Checker 实时校验

## 系统原理

Consistency Checker 在写作过程中实时检查内容一致性，而不是写作后才审查。
遵循「设定即物理」原则——已建立的设定是铁律，不可违反。

---

## 一、实时校验触发时机

### 自动触发

| 触发条件 | 检查内容 | 严重级别 |
|---------|---------|---------|
| 每写完1000字 | 角色状态、等级、物品 | 警告 |
| 每写完一个场景 | 时间线、地点、前后文 | 警告 |
| 每写完一章 | 全部33维度 + 设定一致性 | 错误 |
| 角色升级时 | 力量体系、突破条件 | 错误 |
| 新角色出场时 | 角色档案是否存在 | 警告 |
| 新设定出现时 | 是否与已有设定冲突 | 错误 |
| 伏笔提到时 | 伏笔表是否记录 | 警告 |

---

## 二、校验规则

### 规则1：角色一致性

**检查项**：
- 角色当前等级是否与 `truth-files/characters.md` 一致
- 角色性格是否与宪法定义一致
- 角色持有的物品是否与记录一致
- 角色关系是否与记录一致

**违规示例**：
```
❌ 主角突然从炼气3层变成筑基期（没有突破过程）
❌ 主角原本性格谨慎，突然变得鲁莽
❌ 主角持有的宝物突然出现/消失
```

### 规则2：力量体系一致性

**检查项**：
- 新能力是否在力量体系中已有定义
- 突破是否符合突破条件
- 越级战斗是否合理（有代价或智取）
- 战斗结果是否符合等级差距

**违规示例**：
```
❌ 主角使用了力量体系中不存在的能力
❌ 主角从炼气3层直接突破到筑基期（跳过炼气4-9层）
❌ 主角越3个大境界碾压对手（无合理原因）
```

### 规则3：时间线一致性

**检查项**：
- 当前章节时间是否连续
- 角色年龄、事件时间是否合理
- 季节、天气是否连续
- 事件间隔时间是否合理

**违规示例**：
```
❌ 上一章是春天，下一章变成冬天（没有时间跳跃说明）
❌ 主角闭关修炼3天，出来后世界过了3年
❌ 角色昨天受伤，今天就完全恢复（伤势不合理）
```

### 规则4：世界观一致性

**检查项**：
- 世界规则是否被违反
- 地理环境是否一致
- 势力关系是否变化
- 已建立的历史事件是否一致

**违规示例**：
```
❌ 世界设定人类不能修仙，主角突然修仙
❌ 上一章A城在B城西边，下一章在B城东边
❌ A和B原本是死敌，突然成为盟友（没有过渡）
```

### 规则5：伏笔一致性

**检查项**：
- 新提到的伏笔是否在伏笔表中
- 伏笔状态是否正确（埋设/回收中/已回收）
- 是否有未回收的重要伏笔被遗忘
- 回收的伏笔是否有记录

**违规示例**：
```
❌ 提到「神秘玉佩」，但伏笔表中没有记录
❌ 伏笔H001已回收，但又在后续章节提到
❌ 伏笔H002埋设超过50章未回收
```

### 规则6：因果链一致性

**检查项**：
- 事件是否有原因
- 结果是否符合原因
- 是否有机械降神
- 是否有巧合过度

**违规示例**：
```
❌ 主角突然获得关键道具（没有来源）
❌ 主角在危急时刻突然有人救（之前无铺垫）
❌ 主角连续3次在危机中获得巧合帮助
```

## 三、校验流程

### 写作时实时检查

```
1. AI生成内容
   ↓
2. 提取实体（角色/物品/地点/能力）
   ↓
3. 对照真相文件检查一致性
   ↓
4. 发现问题 → 标记并提示修改
   ↓
5. 无问题 → 继续写作
   ↓
6. 每1000字检查一次
   ↓
7. 章节完成 → 全面审查
```

### 发现问题处理流程

```
1. 发现问题 → 生成警告/错误
   ↓
2. 显示问题详情：
   - 问题类型
   - 问题位置
   - 违反规则
   - 正确做法
   ↓
3. 提供修改建议
   ↓
4. AI自动修复 或 作者手动修改
   ↓
5. 重新检查通过
   ↓
6. 更新真相文件
```

---

## 四、严重程度分级

| 级别 | 名称 | 说明 | 处理方式 |
|------|------|------|---------|
| P0 | 致命错误 | 违反世界观铁律 | 必须立即修改 |
| P1 | 严重错误 | 角色OOC、力量体系违反 | 建议修改 |
| P2 | 一般警告 | 伏笔未记录、小矛盾 | 需要处理 |
| P3 | 建议提醒 | 风格不一致、节奏问题 | 可选修改 |
| P4 | 信息提示 | 实体变更、状态更新 | 确认更新 |

---

## 五、使用指令

```
/nw review consistency  运行一致性检查
/nw review consistency 前后有没有矛盾  自然语言检查
```

## 六、校验报告格式

```
# 一致性校验报告

## 章节：第X章
## 校验时间：[日期]

### 致命错误 (P0)
- ❌ [问题描述] → [修改建议]

### 严重错误 (P1)
- ❌ [问题描述] → [修改建议]

### 一般警告 (P2)
- ⚠️ [问题描述] → [修改建议]

### 建议提醒 (P3)
- ℹ️ [问题描述] → [修改建议]

### 信息提示 (P4)
- 📝 [信息描述] → [确认操作]

---
总计：P0:0 P1:0 P2:0 P3:0 P4:0
状态：✅ 通过 / ❌ 不通过
```
