# 创意约束系统

## 系统原理

创意约束系统通过四重机制防止AI写作时偏离既定设定，确保故事的一致性和创意性。

---

## 一、三轴混搭（Three-Axis Hybrid）

### 原理
每个情节都必须在三个轴向上同时有选择，避免AI单调重复。

### 三轴定义

#### 1. 风格轴（Tone Axis）
| 类型 | 特征 | 示例 |
|------|------|------|
| 轻松 | 幽默、搞笑、日常 | 吐槽、段子、日常互动 |
| 严肃 | 沉稳、克制、深度 | 权谋、哲学、道德困境 |
| 热血 | 激昂、战斗、突破 | 打脸、逆袭、团战 |

#### 2. 冲突轴（Conflict Axis）
| 类型 | 特征 | 示例 |
|------|------|------|
| 内在 | 心理挣扎、道德选择 | 信仰与现实的冲突 |
| 人际 | 角色之间矛盾 | 误会、背叛、信任 |
| 外部 | 环境、敌人、规则 | 怪物追杀、时间限制 |

#### 3. 节奏轴（Pacing Axis）
| 类型 | 特征 | 示例 |
|------|------|------|
| 舒缓 | 日常、探索、对话 | 种田、日常、感情培养 |
| 中等 | 推进、揭秘、铺垫 | 调查、计划、准备 |
| 紧张 | 追逐、战斗、危机 | 生死攸关、倒计时 |

### 混搭规则

每章或每个情节段，必须从三个轴各选一个类型，形成混搭组合：

```
示例混搭：
风格：热血 + 冲突：人际 + 节奏：紧张
→ 热血战斗场景，主角与宿敌对决，节奏紧凑

示例混搭：
风格：轻松 + 冲突：内在 + 节奏：舒缓
→ 轻松日常中，角色内心有挣扎，节奏缓慢
```

### 应用方式

写作时，AI会：
1. 根据章节目标自动选择三轴组合
2. 或根据 `/nw write` 的自然语言描述推导
3. 确保本章不单调重复

### 防重复检查

连续3章不能使用相同三轴组合：
```
第1章：热血 + 外部 + 紧张 ✅
第2章：热血 + 外部 + 紧张 ❌ 重复，需更换至少一个轴
第3章：轻松 + 人际 + 舒缓 ✅
```

## 二、反套路触发器（Anti-Trope Triggers）

### 原理
检测文中出现网文套路时，自动触发反套路改写，防止千篇一律。

### 常见套路库

| 套路 | 检测关键词 | 反套路改写 |
|------|-----------|-----------|
| 退婚流 | 退婚、悔婚 | 改成：对方主动提出，但主角不在乎 |
| 打脸流 | 嘲讽、打脸 | 改成：主角不反击，用实力碾压 |
| 金手指 | 突然获得 | 改成：代价巨大，不是白嫖 |
| 越级挑战 | 越级、碾压 | 改成：付出代价惨胜，或智取 |
| 后宫 | 美女、倾心 | 改成：每个角色都有自己目的 |
| 系统文 | 叮、系统提示 | 改成：系统有阴谋或代价 |
| 老爷爷 | 戒指里、传承 | 改成：老爷爷有目的，不是真心 |

### 触发规则

当检测到套路关键词时：
1. **标记**：识别出使用的是哪种套路
2. **反转**：从反套路库中选择对应的改写方式
3. **应用**：用反套路方式重写该场景

### 示例

```
套路写法：
反派嘲讽主角，然后被主角一巴掌打脸，全场震惊。

反套路改写：
反派嘲讽主角，主角没有反击，只是淡淡看了一眼。
三天后，反派发现自己连主角的影子都追不上。
没有打脸，没有震惊，只有实力差距带来的绝望。
```

## 三、镜像对抗（Mirror Conflict）

### 原理
每个重要冲突中，正反双方必须有对等的合理性，不是简单的善恶对立。

### 应用规则

1. **反派必须有合理动机**
   - 不是纯粹的坏
   - 有自己的信念和目标
   - 从自己的角度看是正义的

2. **主角必须有弱点**
   - 不是无敌的
   - 有自己的盲区和错误
   - 需要付出代价

3. **冲突必须有灰色地带**
   - 不是非黑即白
   - 双方都有自己的道理
   - 读者能理解双方

### 示例

```
普通对抗：
主角正义 vs 反派邪恶

镜像对抗：
主角想保护家园，但必须牺牲少数人
反派想拯救族人，但必须侵占他人土地
双方都正义，但利益冲突不可调和
```

## 四、约束继承（Constraint Inheritance）

### 原理
新章节自动继承前文的约束条件，AI不得随意发明新设定。

### 约束层级

| 层级 | 约束内容 | 来源 |
|------|---------|------|
| L1 | 世界观基本规则 | constitution.md |
| L2 | 力量体系 | power-system.md |
| L3 | 角色当前状态 | truth-files/characters.md |
| L4 | 世界当前状态 | truth-files/current-state.md |
| L5 | 已埋设未回收伏笔 | truth-files/pending-hooks.md |

### 继承规则

1. **不得违反L1-L2**
   - 世界观规则和力量体系是铁律
   - 任何情节必须符合设定

2. **必须考虑L3-L4**
   - 角色不能突然改变状态
   - 世界状态不能跳跃变化

3. **优先处理L5**
   - 新章节应优先推进伏笔回收
   - 不能埋设过多新伏笔而不回收

### 违规检测

写作时自动检测：
```
违规示例：
- 角色突然获得未设定的能力 ❌
- 等级跳跃突破 ❌
- 物品凭空出现 ❌
- 角色性格突然改变 ❌

合规示例：
- 能力在力量体系中已有说明 ✅
- 突破有详细过程和条件 ✅
- 物品有来源说明 ✅
- 性格变化有触发事件和过程 ✅
```

## 使用指令

创意约束已整合到 `/nw review` 指令中：

```
/nw review constraint 查看当前约束
/nw review constraint 检查本章是否违反约束
/nw memory 更新创作宪法添加新约束
/nw review constraint 查看当前三轴混搭组合
```
