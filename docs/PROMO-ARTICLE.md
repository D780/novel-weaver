# NovelWeaver v1.6.0：我给AI做了个网文写作家教，它现在比我还懂怎么写小说

> **TL;DR**: 一个开源的AI网文写作技能，整合22位大神文风库+节奏控制系统+AI味检测+三重记忆，兼容7种AI IDE。不是"AI帮你写小说然后你直接发布"的偷懒工具，而是"AI帮你建立专业写作流程"的辅助系统。

---

## 引子：网文作者的三大灵魂拷问

作为一个写过几百万字网文的作者，我每天打开文档前都要面对三个终极哲学问题：

1. **"今天写啥？"** ——大纲有，但具体到这章怎么写，还是得靠灵感。
2. **"这章是不是太水了？"** ——写完一看，2000字里有800字在描写天气和路人。
3. **"我是不是有AI味了？"** ——用了AI辅助后，读者评论"不禁感到心中涌起一阵复杂的情感"这种句式出现频率太高。

这些问题不解决，写小说就是一场"卡文→硬憋→被吐槽→自我怀疑"的无限循环。

于是我一怒之下，给AI做了个"家教"——**NovelWeaver**，一个全能网文写作助手技能。

现在它不仅能帮我写小说，还能帮我审小说，甚至帮我骂我："你这章太水了，赶紧扩写！"

---

## 一、项目架构：这不是一个Prompt，是一个系统

很多人对"AI写作助手"的理解是：写一段很长的Prompt，然后让AI生成内容。

但NovelWeaver不是这样。它是一个**完整的技能系统**，包含：

```
skill/
├── SKILL.md                    # 主技能文件（指令+规则+工作流）
├── QUICK-REF.md                # 快速参考卡
├── references/                 # 17个专业参考文档
│   ├── rhythm-system.md        # 节奏与情绪控制系统
│   ├── audit-core.md           # 15维度核心审计
│   ├── anti-ai-patterns.md     # 反AI味指南
│   ├── genre-rules.md          # 37种题材写作规则
│   ├── golden-opening.md       # 黄金开篇锻造术
│   └── ...                     # 情绪曲线/记忆系统/写作方法等
├── styles/                     # 22位作者文风库（7大流派）
│   └── authors/
│       ├── 搞笑幽默流/         # 弈青峰/会说话的肘子/三天两觉/卖报小郎君
│       ├── 热血升级流/         # 天蚕土豆/辰东/我吃西红柿/唐家三少/滚开
│       ├── 细腻文青流/         # 猫腻/烽火戏诸侯/愤怒的香蕉/耳根
│       ├── 严谨设定流/         # 爱潜水的乌贼/忘语/言归正传
│       └── ...                 # 特色领域/多神话热血/悬疑推理
├── templates/                  # 14个模板（大纲/章节/角色/场景/伏笔等）
└── scripts/                    # 13个Python预处理脚本
    ├── nw_utils.py             # 公共工具模块
    ├── analyze.py              # 统一分析脚本（三合一）
    ├── chapter_info.py         # 单章结构化提取
    ├── consistency_scan.py     # 一致性扫描
    └── ...                     # 节奏报告/AI味检测/字数检查等
```

**技术亮点**：
- Python预处理脚本将全文(~4000 token)压缩为结构化JSON(~200 token)，**节省约90% token消耗**
- 三重记忆架构：真相文件(设定集) + 长程上下文(前情提要) + 实体关系图(人物关系)
- 兼容7种AI IDE：Trae / Claude Code / Cline / Cursor / Copilot / OpenCode / Continue

### 3. 三重记忆系统：长篇写作不忘事

```
┌─────────────────────────────────────────┐
│           三重记忆架构                    │
├─────────────────────────────────────────┤
│ 第一层：真相文件（Truth Files）           │
│  - 持久化的世界状态和设定                 │
│  - 相当于"设定集"                         │
├─────────────────────────────────────────┤
│ 第二层：长程上下文（Context）             │
│  - 前文摘要和相关章节                     │
│  - 相当于"前情提要"                       │
├─────────────────────────────────────────┤
│ 第三层：实体关系（Entity Graph）          │
│  - 角色关系、地点关联、物品追踪           │
│  - 相当于"人物关系图"                     │
└─────────────────────────────────────────┘
```

**实战效果**：
- 写到第100章，AI还记得主角在第3章捡到的那把剑叫"断水"
- 角色档案自动维护成长轨迹、关系变化、等级进度
- 伏笔追踪表记录埋设/回收状态，避免"只埋不收"

- **网文作者**（全职/兼职）——建立专业写作流程，提升质量和效率
- **想写小说但不知道从哪开始的新手**——6问引导从0到1
- **写长篇容易忘设定的"健忘症患者"**——三重记忆自动维护
- **被读者吐槽"AI味太重"的AI辅助写作者**——AI味检测专治各种AI腔
- **想学习网文写作套路的人**——37种题材规则+22位大神文风库

### ❌ 不适合

- 想"AI一键生成小说然后直接发布"的懒人
- 不愿意花时间完善设定和大纲的人
- 期望AI完全替代创作的人

**定位**：这不是一个"AI帮你写小说"的偷懒工具，而是一个"AI帮你建立专业写作流程，让你写得更好更快更稳"的辅助系统。

## 六、技术实现要点（面向开发者）

### 架构设计

```
用户指令 (/novel-weaver xxx)
    ↓
AI意图识别 + 指令路由
    ↓
自动检索上下文 (RAG)
    ├── truth-files/ (设定)
    ├── novels/ (前文)
    └── .novel-weaver/summaries/ (摘要)
    ↓
Python预处理 (节省token)
    ├── analyze.py --mode single/batch/style
    ├── chapter_info.py --json
    └── volume_batch.py --recent N --json
    ↓
AI生成内容
    ↓
自动审查 (33维度)
    ├── AI味检测 (style_check.py)
    ├── 一致性扫描 (consistency_scan.py)
    └── 节奏分析 (pacing_report.py)
    ↓
更新记忆 + 输出报告
```

### 关键设计决策

1. **Python脚本预处理**：将全文压缩为结构化JSON，降低90% token消耗
2. **三重记忆架构**：真相文件(权威设定) + 长程上下文(前情) + 实体关系(关联)
3. **双轨节奏控制**：S1-S5评级(量化) + 情绪曲线(质化)
4. **向后兼容**：保留旧脚本名称，新脚本作为推荐
5. **多IDE兼容**：通过Skill文件格式适配7种AI IDE

### 扩展性

- 新增文风：在 `styles/authors/` 下添加Markdown文件即可
- 新增题材规则：在 `references/genre-rules.md` 中扩展
- 自定义模板：在 `templates/` 中创建新模板
- 自定义审查规则：修改 `references/anti-ai-patterns.md`
