# 贡献指南

感谢你对 NovelWeaver 的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告 Bug

如果你发现了 Bug，请提交 Issue 并包含以下信息：

1. **简要描述** - 用一句话说明问题
2. **复现步骤** - 详细的操作步骤
3. **预期行为** - 你认为应该发生什么
4. **实际行为** - 实际发生了什么
5. **环境信息** - 操作系统、IDE 版本等
6. **截图** - 如果适用，提供截图

### 提出新功能

如果你想建议新功能，请提交 Issue 并说明：

1. 你想解决什么问题
2. 为什么现有功能不能满足
3. 你期望的行为是什么

### 提交代码

1. **Fork 本仓库**
2. **创建特性分支** - `git checkout -b feature/your-feature`
3. **遵循编码规范**
   - 使用 Markdown 编写所有文档
   - 保持与现有文件一致的格式
   - 中文命名文件，简洁明了
4. **提交变更** - `git commit -m 'feat: 添加你的功能'`
   - 使用 [Conventional Commits](https://www.conventionalcommits.org/) 格式
   - 提交类型：`feat`, `fix`, `docs`, `style`, `refactor`, `test`
5. **推送到分支** - `git push origin feature/your-feature`
6. **创建 Pull Request**

### 添加新文风

如果你想添加新的作者文风：

1. 在 `skill/styles/authors/` 下找到对应的流派目录
2. 如果没有合适的流派，创建新的流派目录
3. 按照现有文风文件的格式编写新作者文风
4. 更新 `skill/styles/author-styles.md` 中的作者列表
5. 在 PR 中说明该作者的代表作和风格特征

### 完善参考文档

如果你想完善参考文档：

1. 找到对应的文档文件
2. 添加或修改内容
3. 确保格式与现有内容一致
4. 在 PR 中说明你的修改内容

## 开发流程

### 本地测试

1. 将 `skill/` 目录复制到你测试项目的 `.trae/skills/novel-weaver/`
2. 在测试项目中使用 `/nw` 指令测试功能
3. 确认修改没有破坏现有功能

### 提交 PR 前检查

- [ ] 确认文件格式正确
- [ ] 确认链接和引用有效
- [ ] 确认没有引入无关文件
- [ ] 确认更新日志已更新（如适用）

## 行为准则

- 尊重所有贡献者
- 接受建设性批评
- 关注问题本身，而非个人
- 保持耐心和友善
